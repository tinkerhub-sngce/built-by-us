# Contributing to Built by Us

> Ship what you build. No matter how small.

---

## About This Project

**Built by Us — Intro Edition** is a space to showcase projects built by students of our campus. This is not about perfection — it's about building, sharing, and learning in public.

---

## Before You Submit

Make sure your project:

- [ ] Is available on GitHub
- [ ] Is built by you or your team
- [ ] Has a proper `README.md`
- [ ] Is forked into the TinkerHub organization

---

## How to Submit

1. **Fork** this repository
2. **Add your project** inside the following structure:

   ```
   built-by-us/
   └── your-project-name/
       └── README.md  [follow the readme template provided]
       └──initiative.png

   ```

3. **Open a Pull Request** with all required details (see below)

---

## Pull Request Requirements

Your PR description must include:

### 1. Project Details
- **Project Name**
- **Short Description** — 1 to 2 lines on what it does
- **Tech Stack** — languages, frameworks, tools used

### 2. Proof of Work
- **GitHub Repository Link**
- **Screenshots / Demo** — images, a video, or a live link

### 3. Ownership
- Confirm the project is built by you or your team
- Mention all contributors

---

## README Guidelines

Your `README.md` inside the project folder must include:

- What the project does
- How to run it locally
- Screenshots or a demo link

---

## Rules

| Allowed | Not Allowed |
|---|---|
| ✅ Half-built projects | ❌ Tutorial-only or copied projects |
| ✅ Experiments & prototypes | ❌ Work that isn't yours |
| ✅ Anything you genuinely built | ❌ Projects without a README |

---

## Review Process

We don't judge perfection. We only check:

- Is it real?
- Is it built by you?

That's it.
